# 02-student.md — Кабинет ученика

## Статус документа

Этап 1: ТЗ для верстки фронта кабинета ученика.

На этом этапе используются временные данные; API и БД не проектируются; экран «Классы» остается заглушкой; виртуальный класс не реализуется.

---

# Анкета ученика после регистрации

После регистрации ученик заполняет базовую анкету. В анкете не выбираются предметы, цели обучения и преподаватели.

Предметы выбираются позже в личном кабинете.

## Поля анкеты

- Фамилия;
- Имя;
- Отчество;
- Год рождения / дата рождения;
- Пол;
- Город / населенный пункт;
- Часовой пояс;
- Кто заполняет анкету: ученик / родитель;
- Фамилия родителя;
- Имя родителя;
- Отчество родителя;
- Телефон ученика;
- Телефон родителя;
- Мессенджер: Telegram / WhatsApp / VK / другое;
- Email.

---

# Родитель

В первой версии отдельного кабинета родителя нет.

Родитель участвует через аккаунт ученика и может контролировать расписание, смотреть дневник, домашние задания, оплаты, писать преподавателю через сообщения и оставить отзыв после установленного количества занятий.

---

# Общий layout кабинета ученика

После заполнения анкеты ученик сразу попадает в личный кабинет.

Слева основной сайдбар:

```text
Фото / имя ученика

Предметы
Классы
Расписание
Домашние работы
Дневник
Материалы
Оплаты
Сообщения
Уведомления
Настройки
```

Раздел «Предметы» открывается первым.

---

# Экран «Предметы»

## Назначение

Экран используется для выбора направления обучения и предметов. Ученик может изменить выбор в любое время.

## Рабочая область

Заголовок: «Предметы».

Блок направлений:

```text
ОГЭ
ЕГЭ
Школьные предметы
Иностранные языки
Начальная школа
```

После выбора направления появляется блок предметов:

```text
Математика
Русский язык
Английский язык
Физика
Информатика
Химия
Биология
История
Обществознание
```

На этапе первой верстки поиск преподавателя отсюда не реализуется.

---

# Экран «Классы»

Статус: заглушка.

```text
Классы

У вас пока нет активных классов.
```

Причина: классы зависят от будущей логики «ученик → преподаватель → предмет → урок → виртуальный класс».

---

# Экран «Расписание»

Показывает расписание ученика на неделю. Ученик не управляет уроками, только просматривает расписание и входит в класс, когда это разрешено.

## Основной вид

```text
Понедельник     15 сентября     2 урока     с 10:00
Вторник         16 сентября     1 урок      с 14:00
Среда           17 сентября     3 урока     с 09:00
Четверг         18 сентября     1 урок      с 12:00
Пятница         19 сентября     2 урока     с 10:00
Суббота         20 сентября     выходной
Воскресенье     21 сентября     выходной
```

## Клик по дню

Открывается список уроков:

```text
10:00–11:00   Математика
15:00–16:00   Английский язык
```

## Клик по уроку

Открывается карточка урока: предмет, преподаватель, дата, время, тема урока, статус.

## Вход в класс

До начала урока: «Класс недоступен». За 10 минут до начала появляется кнопка «Войти в класс». После окончания: «Урок завершён».

## Уведомление

За 10 минут до урока: «Через 10 минут начнётся урок. Математика. Преподаватель Иванов И.И.»

Каналы первой версии: внутри платформы, email.

---

# Экран «Домашние работы»

## Ключевое решение

Источник домашнего задания:

```text
Урок → Класс → Домашнее задание → Ученик
```

Домашние задания не создаются учеником. Они выдаются преподавателем во время урока или после урока через класс.

Раздел «Домашние работы» является агрегатором всех заданий ученика.

## Второй сайдбар

```text
Активные
Просроченные
Проверяются
Проверенные
```

## Рабочая область

Список заданий:

```text
Квадратные уравнения
Математика
Срок: 15 сентября
Статус: В работе
```

## Клик по заданию

Карточка задания: предмет, тема, дата выдачи, срок сдачи, описание задания, прикрепленные материалы, ответ ученика.

Ответ ученика может содержать текст, файл, изображение. Кнопка: «Сдать работу».

Статусы: В работе, Проверяется, Проверено, Требуется доработка, Просрочено.

---

# Экран «Дневник»

Дневник отображает историю обучения ученика: что изучали, что задали, какие материалы использовали, какие рекомендации дал преподаватель.

Дневник не является журналом преподавателя и отображает только информацию, предназначенную для ученика и родителей.

## Второй сайдбар

Предметы ученика:

```text
Математика
Русский язык
Английский язык
```

## Рабочая область

Список уроков:

```text
15 сентября — Квадратные уравнения
22 сентября — Теорема Виета
29 сентября — Графики функций
```

## Клик по уроку

Карточка урока: дата, предмет, преподаватель, тема урока, что изучали, домашнее задание, комментарий преподавателя, материалы урока, оценка если используется.

В дневник ученика не попадают внутренние заметки преподавателя.

---

# Экран «Материалы»

## Общая структура

Верхние табы:

```text
Учебники
Тренажёры
Дополнительные материалы
```

## Учебники

Второй сайдбар: Математика, Русский язык, Английский язык, Физика, Информатика.

Рабочая область: Алгебра 8 класс, Геометрия, ЕГЭ Профиль.

Доступ: Просмотр. Скачивание учебников отсутствует.

## Тренажёры

Второй сайдбар: Математика, Русский язык, Английский язык, Информатика.

Рабочая область: Решу ЕГЭ, Wordwall, LearningApps, SkySmart.

Действие: Перейти.

## Дополнительные материалы

Авторские материалы преподавателей: методички, брошюры, рабочие тетради, сборники задач, шпаргалки.

Статусы:

- бесплатные — Просмотр;
- назначенные преподавателем — Просмотр;
- платные до покупки — название, автор, описание, стоимость, Купить;
- после покупки — Просмотр, Скачать.

Ученик видит бесплатные материалы, назначенные материалы и платные материалы каталога. Доступ к содержимому платных материалов открывается после покупки.

---

# Экран «Оплаты»

Показывает оплаты ученика / родителя: оплачено, предстоит оплатить, история оплат.

Данные: предмет, преподаватель, дата начала обучения, тип оплаты, стоимость урока или курса, статус оплат.

Типы оплаты первой версии: поурочно, курсом. Абонемент зарезервирован на будущее, но скрыт в интерфейсе первой версии.

---

# Экран «Сообщения»

Служебная переписка внутри платформы.

## Верхние табы

```text
Преподаватели
Родители
```

## Второй сайдбар

Список диалогов.

## Рабочая область

Сообщения собеседника слева, свои сообщения справа.

Панель ввода: текстовое поле, смайлы, прикрепить файл, прикрепить изображение, отправить.

Первая версия: текст, смайлы, изображения, файлы. Не входят: голосовые, видеосообщения, звонки.

---

# Экран «Уведомления»

Примеры: через 10 минут урок, проверено ДЗ, добавлен материал, изменено расписание, получено сообщение, получен комментарий, требуется оплата.

---

# Экран «Настройки»

Второй сайдбар: Профиль, Контакты, Безопасность, Уведомления.

Профиль: ФИО, дата рождения, пол, город, часовой пояс, фото.

Контакты: телефон, email, мессенджер, контакты родителя.

Безопасность: сменить пароль, выйти со всех устройств, удалить аккаунт.

Уведомления: внутри платформы, email, Telegram позже.

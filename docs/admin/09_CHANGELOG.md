* административная авторизация;

* Users API;
* просмотр аккаунтов;
* поиск пользователей;
* фильтрация;
* карточка пользователя;

* блокировка;
* разблокировка;
* архивирование;
* восстановление;
* переход в профиль ученика;
* переход в профиль преподавателя.

### Архитектурные решения

Принято

* Users является техническим реестром аккаунтов.
* Анкеты вынесены в отдельные разделы.
* Физическое удаление пользователей запрещено.
* Используется архивирование.
* UI Library отделена от Business Components.
* Каждый новый раздел начинается только после завершения предыдущего.
